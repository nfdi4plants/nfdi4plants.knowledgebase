Here we take a closer look at some typical scenarios of data or sample types that scientists might want to represent in their ARC. With these examples we aim to provide best practices for ARC-supported data annotation allowing to generate machine-readable, interoperable and reproducible data and metadata.

## Request or contribute a use-case

To grow this collection of use-cases, we encourage you to contribute your own use-case or request a use-case that you would like to see covered. To do so, please:

<Steps>

1. Check-out the [ARC Annotation Use-Cases project board](https://github.com/orgs/nfdi4plants/projects/14) for open use-cases
2. If your use-cases is not yet raised, please request it via [issue](https://github.com/nfdi4plants/nfdi4plants.knowledgebase/issues) or contact a data steward directly.
3. Once you want to contribute a use-case, please open a pull request on the [knowledge base repository](https://github.com/nfdi4plants/nfdi4plants.knowledgebase) referencing the related issue.

</Steps>
