What is a pull request, and why is it important for SOP quality?

</ViolaSays>

A pull request is a formal proposal to merge changes into the validated branch.

It enables:

- Side-by-side comparison of versions
- Inline commenting
- Structured discussion
- Transparent decision-making

## Review Questions

When reviewing an SOP change, consider:

- Is the modification scientifically justified?
- Are safety aspects clearly described?
- Are reagent concentrations correct?
- Is terminology consistent?
- Is the change documented in the commit message?

## Benefits of Transparent Discussion

Pull request discussions:

- Document reasoning
- Preserve decision history
- Enable mentorship
- Improve procedural clarity

The review process becomes part of the scientific record.
