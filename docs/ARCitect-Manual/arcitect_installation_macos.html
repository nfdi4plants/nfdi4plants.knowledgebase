ARCitect Installation - macOS

last updated at 2023-09-22

üí° Updated for ARCitect v0.0.8 and higher üí°

  1. You do not need to have ARC Commander installed

üî• Note, that the current .dmg releases are not stable on all macOS systems.

üöß The ARCitect is currently being developed and not fully user-ready, yet. Installation for beta-testing requires an admin account.

Prerequisites Install Node.js

Please install the latest version of Node.js.

  1. Download the installer for your operating system (e.g. macOS Installer)
  2. Once downloaded, open the node-x.x.x.pkg from your downloads folder and follow the "Install Node.js" instructions.
Install Git

Please install and configure Git and Git LFS before using the ARCitect.

Install the ARCitect

  1. Head to the ARCitect's GitHub release page.

  2. Under "Assets" download the latest source code archive (Source code (zip)).

  3. In Finder, open the folder where you downloaded the ARCitect

  4. Unzip the folder (ARCitect-x.x.x.zip). You can move this folder to another location (e.g. Applications).

  5. Open the folder in a terminal (right click ‚Üí services ‚Üí New Terminal at Folder)

  1. In the terminal, check that node.js is installed by executing
node -v

üí° This should display your installed version of node.js, e.g.

v18.17.0

  1. In the terminal, execute the following command to install the ARCitect
npm install

üí° This may take some seconds to minutes depending on your internet connection.

Start the ARCitect
  1. In the terminal, execute the following command to start the ARCitect
npm run debug

üí° A window will pop-up asking whether you want to accept incoming network connections. This needs to be allowed for communication with the DataHUB.

  1. You can leave the terminal running in the background. Once you close the terminal, the ARCitect quits.

  2. Switch to the ARCitect app.

üí° You might have to select the app from your dock.
